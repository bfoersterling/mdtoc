#### tail of a tree?

The tail of a tree is a node with no next sibling, no children, and which \
direct ancestors all have no next sibling.

But the question is if it is even useful to have a reference to the tail of \
a tree.

If you have the reference to the tail then you can directly append to it.\
But there seems to be no situation were you always need to append to the tail.

=> Having a reference to the tail of a tree seems pointless.
